ActionScript 為 Macromedia 公司產品 Flash 之內建語言。Flash 原是針對網路動畫而作，而 ActionScript 的撰寫被整合於時間軸的架構中。 ActionScript 以 JavaScript 為基礎，Processing 則以 Java 為基礎，兩者間有不少雷同。下表列出彼此的比較。

繪圖
ActionScript 在繪圖方面的最佳化做得很成功，在螢幕上繪製形狀的速度較快。因此 Flash 的應用程式能擁有廣大的像素範圍，且對圖形透明化及去鋸齒化的控制力佳。

計算
ActionScript 在計算方面速度較慢，如果應用程式中含有大量元素，Processing 的表現則較優秀。

像素操控
Flash 最初是以向量圖檔的繪製為主去設計的， 所以 ActionScript 缺乏直接控制像素的功能。Processing 則提供各種功能和資料架構，達到輕鬆控制影像像素的功能，造就許多有趣的軟體表現。

3D
ActionScript 裡沒有內建的立體空間表述，Processing 的程式庫則附有內建 3D 引擎。

物件
ActionScript 是物件導向語言，Processing 可以物件導向和程序性兩種方式撰寫。所以初學者可以在了解物件導向的隱喻之前，先行熟悉軟體本身。

您可以在 Macromedia 的網站，獲得有關 ActionScript Reference 的完整資訊。
