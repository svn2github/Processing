總覽

Processing的使用環境整合了程式編輯介面和展示輸出效果的視窗。當您按下播放（Run）按鈕，就會立即執行程式編譯的工作，並將結果展示於視窗中。主要環境的介面上可以執行的功能包括播放（Run）、停止（Stop）、儲存（Save）、開啟檔案（Open）與輸出檔案（Export）。



    



  
  

     

  

    
    
      
座標系統
Processing採用笛卡兒式（Cartesian）座標系統，原點起自於左上角。例如把一個程式的場景設定成寬 320像素、高240像素， [0, 0] 的座標值一定位在左上角，座標值為 [320, 240] 之處則是在最右下角的位置。
      



    



  
  

     

  

    
    
      
由淺入深的三種設定

      
據其難易度，Processing 提供了三種不同的程式撰寫模式，分別為靜態模式、動態模式及Java模式。



	  靜態模式

在靜態模式下，我們可以繪製靜止的圖像。以下的程式會在螢幕上畫出一個黃色的長方形。
size(200, 200);
background(255);
noStroke();
fill(255, 204, 0);
rect(30, 20, 50, 50);


動態模式

在動態模式下，可以選擇是否設定 
      setup() 函式，此函式只會在程式開始時執行一次。 
      loop() 函式則會反覆執行直到程式結束為止。

以下的範例會在視窗中產生一個隨著滑鼠游標移動的長方形（變數 mouseX 和 mouseY 儲存了游標的X和Y座標）。請注意設定背景顏色的函式 background() 出現在 setup() 函式之後的波狀括號內，因為它只需執行一次即可。
void setup()
{
  size(200, 200);
  background(255);
  rectMode(CENTER_DIAMETER);
  noStroke();
  fill(255, 204, 0);
}

void loop()
{
  rect(width-mouseX, height-mouseY, 50, 50);
  rect(mouseX, mouseY, 50, 50);
}


Java模式 （將於Processing 1.0 Beta版發行時推出）

此模式具有最大的彈性，可以在Processing的環境下直接撰寫完整的Java程式。

以下範例由Java語言撰寫，其結果同等於之前的範例。
public class MyDemo extends BApplet {
  void setup()
  {
    size(200, 200);
    background(255);
    rectMode(CENTER_DIAMETER);
    noStroke();
    fill(255, 204, 0);
  }

  void loop()
  {
    rect(width-mouseX, height-mouseY, 50, 50);
    rect(mouseX, mouseY, 50, 50);
  }
}
